Stakeholder Report - Data Integration & Analysis Summary

Executive Summary for Stakeholders

This analysis integrates automotive defect data (100 records) with equipment maintenance data (500 records) to identify failure patterns and cost drivers. The study reveals critical insights for maintenance optimization and quality improvement across multiple equipment categories.

Data Cleaning Process

Comprehensive data preparation was performed across multiple datasets:

Task 1: Steering Defect Data

  • Records processed: 100 records, 52 columns
  • Missing values: Reduced from 178 to 170
  • Cost range validated: $27.69 - $3,205.45
  • Text fields processed: 100% unique customer verbatim

Task 2: Equipment Maintenance Data

  • Work Orders: 500 records, 15 columns
  • Repair Data: 500 records, 13 columns
  • Cost data: Converted to numeric format
  • Date standardization: 2022-04-30 to 2024-02-19

Data Integration Approach

Integration was performed using the explicitly named "Primary Key" field with SO[ORDER]-[SEGMENT] format.

Primary Key Analysis Results

  • Work Orders unique keys: 500
  • Repair Data unique keys: 495
  • Common keys identified: 495
  • Work Orders only: 5 orphaned records

Rationale for Inner Join Selection

Inner Join strategy was selected to ensure complete records with both work orders and repair costs, eliminating incomplete records that could skew analysis and providing a reliable dataset for trend analysis.

3.1 Trend Analysis - Identified Patterns

Cost vs Labor Hours Correlation

Key Finding: Negligible Correlation (r = 0.041)

Analysis reveals virtually no linear relationship between repair costs and labor hours, indicating that cost and time requirements vary independently. Each repair case requires individual resource assessment.
